03 — Data Model

City entities & schema

All spatial data stored as GeoJSON. IDs are human-readable during development (e.g., zone-central, segment-r08). Risk scores normalized to 0–100 across all domains.

Zones

Named geographic areas with sensitivity scores. Types: hospital corridor, school zone, civic center, downtown.

zone_type: hospital_corridor
priority_level: 5
population_sensitivity: 0.92

Road Segments

Individual road sections with lane count, speed limit, baseline capacity, and current congestion state.

baseline_capacity: 1200
lane_count: 3
segment_type: arterial

Facilities

Schools, hospitals, and emergency services — each with priority score and zone membership.

facility_type: hospital
priority_score: 0.98
zone_id: zone-central

Public Buildings

Government offices, libraries, civic buildings — with max kW capacity and load-shedding flag.

max_kw: 520
criticality_level: 3
load_shedding_allowed: true

Incidents

Active events with severity 1–5, affected segment, and real-time cascade tracking.

incident_type: collision
severity: 4
status: active

Emergency Vehicles

Ambulances in transit with destination facility, current ETA, and route segments.

vehicle_type: ambulance
current_status: en_route
eta_minutes: 7.8

Minimum seed data for MVP

  • 4 distinct zones (hospital corridor, school zone, civic center, downtown)
  • 10+ road segments with GeoJSON geometry
  • 1 school (Jefferson Elementary) and 1 hospital (St. Anne) as facilities
  • 3 public buildings with load-shedding enabled
  • 1 ambulance in en-route state
  • 1 seeded collision scenario with full 8-step event timeline
04 — AI & Prediction Engine

Four intelligence layers

A hybrid system: deterministic simulation handles causal state changes, a lightweight forecasting model handles near-term prediction, a rule-based system handles recommendations, and an optional LLM handles explanation. No full ML required for MVP.

L1

Deterministic Event Engine

Simulates what happens when an incident occurs. The most critical layer — all others depend on it.

collision → reduced segment capacity
reduced capacity → congestion increase
congestion → spillback to adjacent segments
spillback → idling pollution increase
congestion → ambulance ETA increase
L2

Forecasting & Prediction

Estimates near-future risk 5–20 minutes ahead. No full ML needed for MVP.

// Methods for MVP forecasting
weighted trend extrapolation
exponential smoothing
moving average delta
gradient-based queue growth model
L3

Recommendation Engine

Maps current and predicted state to ranked interventions. Fully rule-based.

score =
0.30*relevance + 0.25*urgency
+ 0.20*impact + 0.15*confidence
+ 0.10*ease
L4

Explanation Engine

Converts technical outputs to operator guidance. Deterministic templates first; optional LLM rewrite. Must work without LLM.

Styles: operator_brief
executive_summary
judge_demo_narration
public_alert_simulation

Risk domain formulas

Traffic Risk

traffic_risk = clamp(
0.35*congestion_index
+ 0.25*queue_pressure
+ 0.20*speed_drop
+ 0.20*spillback_factor, 0, 100)

Emergency Delay Risk

emergency_risk = clamp(
0.45*eta_delta_score
+ 0.25*corridor_congestion
+ 0.20*blocked_route_factor
+ 0.10*hospital_criticality, 0, 100)

Air Quality Risk

air_quality_risk = clamp(
0.40*aqi_score
+ 0.20*pm25_score
+ 0.20*traffic_density_score
+ 0.20*sensitivity_multiplier, 0, 100)

Composite Unified Risk

unified_risk =
0.35*traffic
+ 0.25*air_quality
+ 0.25*emergency_delay
+ 0.15*energy_strain

05 — API Reference

REST endpoints

Base URL: http://localhost:8000/api/v1

All responses use snake_case. Risk scores normalized to 0–100. Timestamps ISO 8601. Spatial payloads GeoJSON. Real-time updates via SSE at /stream/dashboard.

Core

GET/healthService health check
GET/dashboard/overviewTop-level city state, risk score, headline, active incidents
GET/map/layersAll renderable entities with GeoJSON geometries

Simulation

POST/simulations/startStart named scenario, returns scenario_run_id
POST/simulations/stepAdvance simulation one event step
POST/simulations/playBegin timed playback with speed_multiplier
POST/simulations/pausePause timed playback
POST/simulations/resetReset scenario to baseline state
GET/simulations/{scenario_run_id}Current scenario state and event timeline

Incidents

GET/incidentsList incidents. Filter: status, severity_gte, zone_id
POST/incidentsCreate a manual incident with type, severity, segment, location
GET/incidents/{incident_id}Detailed incident record

Risk

GET/risk/summaryCity-wide risk score, per-domain breakdown, top stressed zones
GET/risk/zones/{zone_id}Zone-level risk details and dominant factor
GET/risk/corridors/{corridor_id}Corridor-level risk for grouped road segments

Recommendations & Actions

GET/recommendationsRanked recommendations. Filter: incident_id, zone_id, status
POST/recommendations/recomputeForce recomputation after simulation state change
POST/actions/applyApply an action and receive expected_effect deltas
GET/metrics/impact-summaryBefore/after metrics for current scenario state

Explanations

POST/explanations/generateGenerate natural language summary in specified style
GET/stream/dashboardServer-Sent Events for live state updates

Example: POST /actions/apply

Request

{
"action_type": "ambulance_green_wave",
"target_entity_id": "corridor-hospital-east"
}

Response

{
"status": "applied",
"expected_effect": {
"queue_length_delta_pct": -19,
"ambulance_eta_delta_min": -1.3
}
}

07 — Demo Scenario

School Corridor Collision

The primary demo scenario. A two-vehicle collision on Central Ave triggers a cascade across all five city systems. The full demo runs 3–4 minutes and is designed for judges to understand the value within 60 seconds.

Event sequence (timed)

T+0s — Baseline

City operating normally

Risk score: 26. Traffic stable, AQI normal, all emergency routes clear, building loads within range.

T+8s — Trigger

Collision created on Central Ave

Severity 4 incident. Two vehicles block one lane on segment-r08, downstream from the hospital corridor. Red marker appears on map.

T+16s — Cascade 1

Traffic spillback begins

Speed drops 26%. Queue grows 34%. Segments r08, r09, r10 all affected. Road layer shifts amber to red.

T+24s — Cascade 2

School zone AQI warning triggered

AQI reaches 112. PM2.5 at 39.4 µg/m³. Jefferson Elementary zone turns amber on the map.

T+32s — Cascade 3

Ambulance ETA increases 3.4 min

Ambu-01 en route to St. Anne Hospital. ETA grows from 7.8 to 11.2 minutes. Emergency delay risk score hits 89.

T+40s — Intelligence

4 recommendations generated

Ranked by urgency and impact: (1) ambulance green wave, (2) reroute general traffic, (3) school zone air alert, (4) reduce non-critical HVAC load.

T+52s — Intervention

Actions applied

All four interventions activated. City state begins recovering. Queue animations reverse. Ambulance route line brightens.

T+64s — Impact

Before/after metrics available

Ambulance ETA −38%. Queue −19%. Exposure risk −21%. Energy shifted 74 kW from non-critical buildings.

08 — Engineering Roadmap

Build order & milestones

Build in layers, not all at once. Correct order: believable data model — good-looking frontend shell — deterministic simulation — risk engine — recommendations — polish. Do not start backend-heavy abstractions before the map renders.

Milestone 1

Repository & Local Environment

Monorepo setup. Frontend and backend boot. Postgres and Redis run locally. Sample data loader works.

Milestone 2

Static City Map

Map page with zones, roads, facilities, and buildings rendered. Mock summary bar. Layer controls. Seed data loaded.

Milestone 3

Dynamic Snapshots & Metrics

Traffic, AQI, and energy snapshot endpoints. Top summary metrics. Mini charts. The city feels alive before incidents exist.

Milestone 4

Incident Simulation Engine

Scenario loader. Start / step / play / pause / reset endpoints. Road segment state mutation. Incident marker. Timeline UI.

Milestone 5

Risk Engine

All four risk models implemented. Unified city risk score. System can explain severity numerically across domains.

Milestone 6

Recommendation Engine

Action catalog. Rule-based recommendation logic. Ranking and confidence scoring. Impact estimates. Apply action endpoint.

Milestone 7

Explanation Layer

Deterministic summary templates. Optional LLM summary endpoint. Explanation panel UI. Robust fallback if model unavailable.

Milestone 8

Demo Polish

Presentation mode. Animations and transitions. Before/after cards. Smooth event sequence. Strong opening and closing state.

09 — Testing & Quality

Testing, observability & security

Primary goal: prevent broken demo flows. Secondary goal: make simulation deterministic. All risk formulas and recommendation rules must be unit-testable before building the UI that depends on them.

Backend — Unit Tests

  • Risk scoring formulas for all 4 domains
  • Recommendation ranking logic
  • Action impact estimator deltas
  • Simulation step state mutations
  • Explanation template generation

Backend — Integration Tests

  • Scenario start and reset endpoints
  • Risk recomputation after incident
  • Recommendation generation after risk change
  • Action apply flow end-to-end
  • Impact summary endpoint

Pre-demo checklist

Run through this before every presentation. Map tile loading and the explanation panel are most likely to fail in live demos — check both twice.

  • Run full scenario once locally end-to-end
  • Verify Mapbox tiles load completely
  • Verify seed data present — run /health and /dashboard/overview
  • Verify backend health endpoint returns 200
  • Verify explanation panel fallback works (disable LLM key temporarily)
  • Verify before/after cards update after actions applied
  • Verify scenario reset returns to clean baseline state
  • Have static screenshots ready as fallback
